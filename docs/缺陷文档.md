## 二、新发现缺陷

### BUG-008：审计取证单导出模板名引用错误（致命）
- **严重程度**：严重
- **优先级**：P0
- **所属模块**：审计取证单 — StageEvidence.vue
- **关联用例**：TC-EVD-004
- **需求来源**：审计流程4.30 第9步 + 系统设计对照与更新.md §5 模板清单
- **复现步骤**：
  1. 创建项目 → 进入审计取证单步骤
  2. 填写取证单信息并保存
  3. 点击"导出文档"按钮
- **预期结果**：成功导出 `14审计取证单.docx` 模板填充后的文档
- **实际结果**：导出失败，提示"模板文件不存在：11审计取证单"

- **修复建议**：将 `'11审计取证单'` 改为 `'14审计取证单'`
- **状态**：✅ 已修复 — `StageEvidence.vue:354` 已更新为 `'14审计取证单'`

### BUG-010：审计底稿未从取证单自动导入项目名称（一般）
- **严重程度**：一般
- **优先级**：P1
- **所属模块**：审计工作底稿 — StageWorkingPaper.vue
- **关联用例**：TC-WP-002, TC-FORM-004
- **需求来源**：审计流程4.30 "取证单中的项目名称导入到审计工作底稿中" + 各页面关系.docx 第11条
- **复现步骤**：
  1. 创建项目，保存审计通知书
  2. 保存审计取证单
  3. 从项目详情页直接进入"审计工作底稿"步骤（非从取证单跳转）
  4. 点击"新增底稿"
- **预期结果**：底稿表单的"项目名称"字段自动从前序数据（取证单/通知书）导入
- **实际结果**：底稿表单的"项目名称"为空，需要用户手动输入
- **根因分析**：StageWorkingPaper.vue 作为专用组件（LIST_STAGES），不实现 GenericStageForm 的 `autoImportFromPreviousStages` 逻辑。虽然 WORKFLOW_STEPS 配置了 `importFrom: ['evidence']`，但底稿页面不使用此配置。`resetForm()` 仅从 `props.projectInfo.name` 获取项目名称，但当用户直接从项目详情进入底稿页时，不会自动加载前序步骤数据。从取证单跳转时（`fromEvidence=1`），只传递 `matterContent` 和 `matterIndex`，不传递项目名称。
- **修复建议**：
  1. 在 `resetForm()` 中始终设置 `projectName: props.projectInfo?.name || ''`（当前已实现 ✅）
  2. 在 `onMounted` 中加载前序步骤（取证单）数据，自动填充 `auditMatter` 等字段
  3. 从取证单跳转时增加传递 `projectName` 参数
- **状态**：✅ 已修复 — 取证单跳转时增加传递 `projectName` 参数，底稿页面接收并预填
- **备注**：`resetForm()` 中已有 `projectName: props.projectInfo?.name || ''`，项目名称可从项目信息获取。取证单跳转时现已传递 `projectName` 参数。

### BUG-011：审计底稿"审计认定的事实摘要"不导入审计报告（一般）
- **严重程度**：一般
- **优先级**：P2
- **所属模块**：审计报告 — GenericStageForm.vue + stageConfigs.ts
- **关联用例**：TC-FORM-004
- **需求来源**：审计流程4.30 "底稿中的审计认定的事实摘要及结论按顺序号导入到审计报告中的发现的主要问题和整改要求" + 各页面关系.docx "审计底稿的内容从取证单中导入"
- **复现步骤**：
  1. 完成审计底稿（填写"审计认定的事实摘要"和"审计结论"）
  2. 进入"审计组审计报告"步骤
  3. 检查"审计发现的主要问题"字段是否自动填充
- **预期结果**：审计报告中"problemsFound"字段自动聚合所有底稿的"factSummary"和"auditConclusion"
- **实际结果**：审计报告不会自动聚合底稿内容（autoImport 仅匹配同名字段，report 的 `problemsFound` 与 working_paper 的 `factSummary` 不同名）
- **根因分析**：GenericStageForm 的 autoImportFromPreviousStages 函数仅按同名字段匹配导入。report 步骤配置了 `importFrom: ['survey', 'evidence', 'working_paper']`，但 report 的字段名（如 `problemsFound`）与前序步骤的字段名（如 `factSummary`）不同，无法自动匹配。需要跨表聚合逻辑。
- **修复建议**：在 report 阶段的 GenericStageForm 中添加专用的聚合导入逻辑：查询所有底稿记录，按索引号排序，将 `factSummary` + `auditConclusion` 拼接为 `problemsFound` 的默认值。
- **状态**：✅ 已修复 — GenericStageForm.vue 的 onMounted 中新增跨表聚合逻辑，查询 working_papers 表按索引号排序拼接 factSummary+auditConclusion 到 problemsFound
