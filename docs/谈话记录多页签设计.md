# 谈话记录多页签设计文档

## 概述

将审计谈话记录阶段从单一文本编辑器改为 4 个页签的多模版填报组件，每个页签对应一个独立模板，支持独立填写、保存和导出。

## 适用范围

- 阶段：`interview_record`（谈话记录）
- 审计类型：经济责任审计

## 页面结构

```
interview_record「谈话记录」
├── Tab: 调查谈话笔录    对应模板 13（13调查谈话笔录模板.docx）
├── Tab: 述职报告提纲    对应模板 15（15述职报告提纲.docx）
├── Tab: 民主测评(旗直)  对应模板 17（17民主测评表旗直领导干部.doc）
└── Tab: 民主测评(乡镇)  对应模板 18（18民主测评表乡镇政府领导干部.doc）
```

## 数据流

```
notice 阶段保存的 auditedLeaderName
        │
        ▼ 自动引用（importFrom: ['notice']）
┌───────────────────────────────────────────────┐
│  Tab 1: 述职人  ← auditedLeaderName          │
│  Tab 2: **同志  ← auditedLeaderName          │
│  Tab 3: **同志  ← auditedLeaderName          │
└───────────────────────────────────────────────┘
        │ 用户填写各 Tab 表单
        ▼
┌───────────────────────────────────────────────┐
│  保存：嵌套 JSON → stage_progress.data_json  │
│  导出：各 Tab 表单字段 → 对应模板占位符       │
└───────────────────────────────────────────────┘
```

### Tab 3：民主测评（旗直）（模板17）

**纯导出模式，无输入框。** 占位符自动从审计通知书引用。

| 占位符 | 数据来源 |
|--------|----------|
| `{auditedLeaderName}` | 通知书 → auditedLeaderName（被审计领导干部姓名，即 `**同志`） |

### Tab 4：民主测评（乡镇）（模板18）

**纯导出模式，无输入框。** 同上，占位符自动从通知书引用。

导出时自动读取 notice 阶段的 `auditedLeaderName` 注入模板，用户无需手动输入，直接点击「导出文档」即可。

## 模板占位符

所有 4 个模板均使用 `{content}` 作为主占位符。导出时表单字段按 `label：value` 格式拼成结构化文本，一次性填入 `{content}`。

| Tab | 模板文件 | 导出文件名 | 占位符 |
|-----|---------|-----------|--------|
| 调查谈话笔录 | 13调查谈话笔录模板.docx | 调查谈话笔录.docx | `{content}` |
| 述职报告提纲 | 15述职报告提纲.docx | 述职报告提纲.docx | `{content}` |
| 民主测评(旗直) | 17民主测评表旗直领导干部.docx | 民主测评表(旗直).docx | `{content}` |
| 民主测评(乡镇) | 18民主测评表乡镇政府领导干部.docx | 民主测评表(乡镇).docx | `{content}` |

> **注意**：原模板 17/18 为 .doc 格式，已创建同名的 .docx 最小版本。用户需将原 .doc 文件格式化为 .docx 并替换，以保留原始版式。
