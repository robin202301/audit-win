## 二、当前实现架构总览

### 2.1 技术栈

- **主进程**: Electron + TypeScript (tsc && tsc-alias 编译)
- **渲染进程**: Vue 3 (Composition API) + Vite + Pinia + Vue Router
- **数据库**: SQLite (better-sqlite3)
- **文档处理**: docxtemplater + pizzip（模板填充）、mammoth.js（Word 解析）
- **打包**: electron-builder → release/ 目录

### 2.2 审计类型（仅三种）

1. **经济责任审计**
2. **预算执行审计**
3. **专项审计调查**

### 2.3 项目结构

```
audit-win/
├── src/
│   ├── main/
│   │   ├── index.ts
│   │   ├── ipc/
│   │   └── services/templateService.ts
│   ├── renderer/
│   │   ├── views/
│   │   │   ├── Home.vue              # 项目列表（含新建项目，审计类型下拉选择）
│   │   │   ├── ProjectDetail.vue     # 项目详情 + 5 阶段 Tab + 25 步骤列表

│   │   │   ├── StageEvidence.vue     # 取证单 1:N 管理（专用）

│   │   │   └── stages/
│   │   │       ├── stageConfigs.ts         # 24 步骤表单配置
│   │   │       ├── GenericStageForm.vue    # 通用结构化表单
│   │   │       └── UniversalStageEditor.vue # 通用文本编辑器
│   │   └── stores/project.ts
│   ├── database/
```

### 2.4 数据库表

| 表名 | 用途 |
|------|------|
| `projects` | 审计项目（name, audited_target, audit_type） |
| `stage_progress` | 阶段进度（25 行/项目，project_id + stage + data_json） |
| `evidence_items` | 取证单（独立表） |
| `working_papers` | 底稿（独立表） |

- **25 步全覆盖**：所有步骤均有对应的 GenericStageForm 或专用页面实现
- **数据引用链**：importFrom 配置已按各页面关系.docx 要求修正
