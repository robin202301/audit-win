## 二、当前实现架构总览

### 2.1 技术栈

- **主进程**: Electron + TypeScript (tsc && tsc-alias 编译)
- **渲染进程**: Vue 3 (Composition API) + Vite + Pinia + Vue Router
- **数据库**: SQLite (better-sqlite3)
- **文档处理**: docxtemplater + pizzip（模板填充）、mammoth.js（Word 解析）、xlsx（Excel 处理）
- **打包**: electron-builder → release/ 目录

### 2.2 审计类型（仅三种）

1. **经济责任审计**
2. **预算执行审计**
3. **专项审计调查**

### 6.5 技术实现细节（更新于 2026-05-28）

#### 6.5.1 编辑/只读状态切换方案

**问题**：HTML `readonly` 属性在 Chrome/Edge 浏览器中动态切换时存在兼容性问题，修改 readonly 从 true 到 false 不总能恢复输入框可编辑状态。

**解决方案**（`GenericStageForm.vue` + `UniversalStageEditor.vue`）：
- `isEditing` 默认值为 `true`（首次进入即为编辑态）
- 使用 `v-if="isEditing"` / `v-else` 完全分离两个 DOM 容器
- 编辑态：`<input v-model="formData[field.key]">` 或 `<textarea>`
- 只读态：`<div class="gov-input-display">{{ displayValue(field) }}</div>`
- 配合 `:key="'edit-' + editKey"` 和 `:key="'view-' + editKey"` 强制 Vue 重建 DOM
- 点击"修改"按钮：`isEditing = true; editKey++`
- 点击"保存"成功后：`isEditing = false`

#### 6.5.2 模板占位符提取修复

**问题**：Word 经常把 `{xxx}` 占位符拆分成多个 XML run（如 `<w:t>{</w:t>` + `<w:t>xxx</w:t>` + `<w:t>}</w:t>`），直接在 XML 上正则匹配会提取到垃圾内容或遗漏占位符。

**解决方案**（`templateService.ts` — `getTemplatePlaceholders`）：
1. 先 `xml.replace(/<[^>]+>/g, '')` 去除所有 XML 标签获取纯文本
2. 再从纯文本中用正则 `/\{([^{}]+)\}/g` 匹配占位符

#### 6.5.3 模板路径解析修复

**解决方案**（`templateService.ts`）：
- 提取共享函数 `getTemplateBasePath()`，三处路径解析共用
- 开发模式优先使用 `__dirname` 相对路径，回退尝试 `app.getAppPath()`
- 增加目录模糊匹配：遍历文件名比对（处理路径偏移问题）

#### 6.5.4 日期格式化

所有导出文档中的日期字段通过 `formatDateToChinese()` 函数转换为中文格式：
- 输入：`2026-01-01` → 输出：`2026年1月1日`

#### 6.5.5 日期校验规则（来自审计流程4.30）

**三层日期校验**（`GenericStageForm.vue`）：
1. **前置校验**：必须先保存审计通知书，才能开启后续阶段（`checkNoticeSaved`）
